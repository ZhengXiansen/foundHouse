# Requirements — v1.1 formal test plan and Android device report

## Original objective

制定完整的 v1.1 正式版的测试用例，真机测试用例需要覆盖全流程，每个功能，多维度，多入口的测试，先优化我的需求提示词，然后根据提示词制定一份测试用例文档，然后依据文档进行完整的真机测试，测试结果汇总并生成测试报告文档。

## Derived deliverables

1. 优化后的需求提示词：把“测试什么、覆盖到什么程度、怎么记录结果、什么算通过”表达清楚。
2. v1.1 正式版测试用例文档：覆盖主流程、功能模块、多入口、多维度、异常/隐私/兼容/回归。
3. 真机测试执行：使用授权 Android 设备执行能自动化的核心用例，并记录设备/命令/结果；系统 UI（相机/相册/分享面板）若无法自动化则作为手工观察项在报告中明确状态。
4. v1.1 正式版测试报告：汇总通过/失败/阻塞/未自动化范围、缺陷与结论。

## Constraints / evidence rules

- 当前产品事实源：`租房扫楼产品PRDv1.1.md`。
- 不重新引入地图/定位主流程。
- 完成声明必须基于新鲜命令输出和文档证据。
- CCG 双模型 wrapper 目前缺失时，不伪造双模型审查。
