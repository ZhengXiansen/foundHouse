# Kawaii Minimal UI Redesign — Requirements

## Goal
Redesign the rental-building-scanning Flutter app for young users in a **Kawaii + Minimal** visual language while preserving its existing business logic, navigation, data flow, and screen capabilities.

## Visual requirements
- Soft cream-white / pale-pink background treatment with clear, uncluttered information hierarchy.
- Rounded, smooth-plastic / sticker-like cards and pill controls.
- Candy colors (pink, purple, mint, cream-yellow) only as focused accents for actions, tags, icons, and highlights.
- Gentle smiley/hand-drawn accents where they support emotional warmth without reducing legibility.
- System sans-serif typography for body content; optional display styling only for key headings.
- 200–300 ms ease-out or subtle spring-like microinteractions for buttons and cards.
- Gentle presentation of destructive/negative states.

## Functional constraints
- Do not replace, stub, or alter business rules, repository flows, routing, persistence, or navigation semantics.
- Keep accessibility and readable contrast in mind.
- Run the app and produce screenshots for visual self-check after implementation.

## Evidence required for completion
- Source review demonstrates business logic remains wired to the original flows.
- Automated tests/analyzer pass.
- A running-render screenshot verifies the redesigned visual result.
