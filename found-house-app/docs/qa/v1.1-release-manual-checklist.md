# v1.1 发布前人工检查清单

> 适用版本：v1.1 手动村 / 楼栋 / 房源扫楼流程  
> 依据文档：`租房扫楼产品PRDv1.1.md`、`docs/qa/v1.1-test-cases.md`、`docs/qa/v1.1-android-device-test-report.md`  
> 目标：在自动化测试通过后，用一台真实 Android 手机补齐系统 UI 与发布体验验收，确认可以进入发布候选状态。

---

## 0. 发布判断口径

### 0.1 可以发布的最低条件

发布前必须同时满足：

- [ ] 自动化准入命令全部通过，且记录命令输出。
- [ ] Android 真机可以安装、冷启动、热启动、退出重进。
- [ ] v1.1 主流程可完整跑通：新增村 → 进入村 → 新增楼栋 → 在楼栋下快速记录房源 → 补全详情 → Checklist / 风险 → 对比。
- [ ] 自动化未覆盖的 3 类系统 UI 人工通过：拍照、相册、系统分享 / 导出。
- [ ] App 用户主流程不再暴露或依赖地图 / 定位 / 第三方地图 API。
- [ ] 默认导出不泄露联系人、门牌、详细地址等敏感信息。
- [ ] 没有 P0/P1 阻断缺陷。

### 0.2 一票否决项

出现任一情况，不建议发布：

- [ ] App 无法安装、启动，或主流程中出现崩溃 / 白屏 / 卡死。
- [ ] 新增村、楼栋、房源任一核心记录无法保存或退出重进后丢失。
- [ ] 快速记录可以在未选择村的情况下创建房源。
- [ ] 首页或主路径重新出现 `扫楼地图`、定位失败、经纬度、地图 API、高德等地图优先体验。
- [ ] 拍照 / 相册授权、取消或成功选择路径导致崩溃。
- [ ] 对比导出默认泄露联系人、门牌、详细地址。
- [ ] 发现真实设备上有明显性能问题：连续操作 3 分钟内频繁卡死、无法响应或异常退出。
