## 5. 关键验收点明细

### 5.1 v1.1 主流程切换

- 默认启动进入「首页」，不是地图页。
- 一级 Tab 为「首页 / 房源 / 对比 / 我的」。
- 首页空态展示「还没有村」「新增村」。
- 已有村时展示「继续扫楼」「村列表」与统计。
- 主路径断言未出现：`扫楼地图`、`定位失败`、`经纬度`、`高德`、`第三方地图 API`。

状态：**PASS**。

### 5.2 Village -> Building -> House 归属

真机脚本覆盖：

- 首页新增村「白石洲」。
- 村详情新增「1号楼」。
- 从楼栋卡片「在此楼记录房源」进入快速记录，保存后断言：
  - `house.villageId == village.id`
  - `house.buildingId == building.id`
- 从首页村卡片/继续扫楼入口进入快速记录，保存后断言：
  - `house.villageId == baishizhouId`
  - `house.buildingId == null`
- 从村详情「记录房源」进入快速记录，保存后断言：
  - `house.villageId == shangshaId`
  - `house.buildingId == null`
- 直接打开 `/scan/quick-record` 且无 `villageId` 时，断言不创建房源并提示先选择村。

状态：**PASS**。

## 7. 缺陷与风险

### 7.1 产品缺陷

- 未发现 P0/P1 产品缺陷。

### 7.2 测试脚本修复

| 问题 | 影响 | 处理 |
| --- | --- | --- |
| `full_device_flow_test.dart` 在真机 route replacement 退出动画期间过早断言旧「快速记录」消失 | 造成误报失败，非产品问题 | 在进入「房源详情」后额外 pump 若干帧再断言 |
| formal 脚本 `ensureVisible` 遇到多匹配 Finder，以及列表滚动后同 route 未复位 | 造成误报失败，非产品问题 | 使用 `finder.first`；需要回到顶部时反向滚动查找 |

### 7.3 残余风险

- 手工系统 UI 项未执行：相机、相册、系统分享面板需发布前人工补测。
- 底层地图/AMap/MapSnapshot 代码仍按 PRD v1.1 Phase 2 策略保留；本轮只验证主流程不暴露/不依赖。
- 旧文档仍可能保留 map-first 历史说明；README/CLAUDE 已有状态标注，后续 Phase 2 可统一清理。
