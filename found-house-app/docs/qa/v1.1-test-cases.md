## 1. 范围与优先级

| 优先级 | 定义 | 发布要求 |
| --- | --- | --- |
| P0 | 阻断 v1.1 核心闭环或可能造成数据归属错误/隐私泄露 | 必须通过或修复 |
| P1 | 影响主要功能完整性、多入口、多维度验收 | 正式版应通过 |
| P2 | 边界体验、系统 UI、低频入口或可延期增强 | 可记录为发布前补充/后续优化 |

## 2. 自动化与手工边界

| 类型 | 说明 | 证据 |
| --- | --- | --- |
| Static | `flutter analyze`、文档/旧文案检查 | 命令输出 |
| Unit/Widget | `flutter test`，含 repository/widget 回归 | 命令输出 |
| Device-Auto | Android 真机 `flutter test -d <device> integration_test/...` | 命令输出 + 断言 |

## 5. 发布判断口径

- 若 `flutter analyze`、`flutter test`、两个 Android 真机 integration 脚本均通过，可判定：v1.1 App 内核心闭环具备正式版发布基础。
- 若 Manual-System 项未执行，只能判定「App 内可自动化核心闭环通过」；相机/相册/系统分享仍需发布前人工补测。
- 若任何 P0 失败，应标记为阻断，不建议发布。
